Apuntes de Git

Git Estándar

git init ↦ Nos permite inicializar el registro de git para una carpeta.

git add archivo o etiqueta ↦ Nos permite indicar que archivos se agregaran o deben ser actualizados para el próximo commit.

git status ↦ Nos permite inicializar el registro de git para una carpeta.

git commit -m "datos" ↦ Nos permite inicializar el registro de git para una carpeta.

git checkout [objeto commit o nombre de rama] ↦ nos permite viajar "en el tiempo" o entre ramas del repositorio.

git branch ↦ nos permite ver las ramas existentes en nuestro proyecto.

git clone ↦ nos descargar el repositorio a nuestro pc.

git log ↦ nos muestra un registro detallado con los commits y notas

git reset[--soft] o [--hard] ↦ el primero borrara el objeto commit, el segundo borra el objeto y los datos en pc.

git merge[rama] integra la rama objetivo al HEAP (posición actual)


Git Notes

Nos permite agregar notas a nuestros commits sin afectar al mismo

git notes add -m "datos" ↦ Nos permite notas a nuestros commits.

git notes list ↦ Nos lista todos los objetos commit con notas sintaxis [objeto commit] [objeto nota].

git notes append -m "datos" [objeto nota] ↦ Nos permite agregar información al final de un objeto nota que exista o lo crea.

git notes remove [objeto commit] ↦ nos permite borrar todos los objetos nota de un objeto commit.


Apuntes GitHub

¿Qué es GitHub?

Es una plataforma online donde los programadores suben sus proyectos, que dentro de la plataforma se llaman repositorios.

También nos permite tener una forma gráfica para ver la evolución de nuestro proyecto.

Conceptos

Fork ↦ Crear una copia de un repositorio ajeno en nuestro perfil para trabajarlo

PullRequest ↦ Si hemos realizado cambios podemos ofrecerlos al dueño para que actualice su proyecto.

Issues ↦ Es la forma en que los usuarios que interactúen con el proyecto pueden notificar sus problemas o pedir cambios en el mismo, funciona como un comentario/foro para que el/los desarrolladores pueden mantener actualizado el proyecto o estar en contacto con los consumidores.

Comandos para relacionar Git con GitHub

Git remote add"nombre" "url del proyecto" ↦ Crea un enlace que esta vinculado a la url y se llama "nombre" (lo normal es "origin").

Git remote remove"nombre" ↦ Elimina el enlace elegido.

Git push"origin" "rama" ↦ Sube el HEAD usando el enlace a la rama indicada (de no indicarse tomara la rama en que esta)

Git fetch"nombre del enlace" ↦ Sirve para actualizar tu version de proyecto desactualizada. [Esto es util si trabajas en tu pc y alguien actualiza el proyecto]
