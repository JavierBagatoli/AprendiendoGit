Contról de versiones

El eje de esta herramienta es el control de versiones, cada cierto tiempo es posible que debamos encontrar las diferencias entre versioens o encontrar en que version especifica se añadio algun comportamiento a nuestro proyecto

git bisect

Uso

git bisect start ↦ Comenzara el estado de comparacion de git

git bisect new["commit del proyecto"]

git bisect old["commit del proyecto"]

Para el caso en que nos llege a confundir los terminos new o old existe

git bisect start --term-old "nombre" y --term-new "nombre"

git bisect skip ↦ Nos permite saltar una etapa de comparacion.

git bisect reset ↦ Dara por concluida la exploracion y nos regresara al HEAD.

Fusión de Ramas

En cada proyecto llega el momento se han solventado todos los problemas que han abierto ramas y toca unir las estas a la principal para ello contamos con:

git merge[Nombre de la que voy a unir a la en que estoy] ↦ Une ambas ramas y permite ver los commits de forma paralela

git rebase [Nombre de rama que se unira a que estoy] ↦ Pone la rama actual al final de la seleccionada dejando un log de commits lineal

git rebase -i "commit~ numero de versiones anteriores" ↦ Al igual que git rebase xxx permite un log de commits lineal pero en este caso sobrescribe a partir del elegido (los commit posteriores a ese punto seran los de nuestra rama)
